# 複数タブでテストする方法

## 新しい仕様（性善説ベース）

このアプリは完全な性善説ベースで設計されているため、**もうデバッグモードは必要ありません**。

## 仕組み

- **ユーザー名がそのままuser_idとして使われます**
- 同じ名前を入力すれば、別のタブでも「同じプレイヤー」として認識されます
- LocalStorageやSessionStorageでのユーザーID管理は行いません

## 複数タブでテストする手順

### 1人でディーラーとプレイヤーをテストする場合

**ただしそれぞれ異なる名前を入力するだけです！**

1. **タブ1（ディーラー）**
   ```
   http://localhost:5173/
   ```
   - ユーザー名：「dealer1」
   - 合言葉：「test1」
   - ディーラーとして入室

2. **タブ2（プレイヤー1）**
   ```
   http://localhost:5173/
   ```
   - ユーザー名：「player1」
   - 合言葉：「test1」
   - プレイヤーとして入室

3. **タブ3（プレイヤー2）**
   ```
   http://localhost:5173/
   ```
   - ユーザー名：「player2」
   - 合言葉：「test1」
   - プレイヤーとして入室

### 同じプレイヤーを複数タブで操作する

**同じ名前を使えば、複数タブで同じプレイヤーを操作できます。**

例：
- タブ1で「player1」としてログイン
- タブ2でも「player1」としてログイン
- → 両方のタブが同じプレイヤーとして認識されます

これは**性善説ベース**の設計であり、デバッグやテストを簡単にするための仕様です。

## 注意事項

- 同じ名前を使うと「プレイヤーの乗っ取り」が可能です（意図的な仕様）
- 本番環境では、プライベートな環境での使用を推奨します
- 悪意のあるユーザーによる不正操作は防げません
